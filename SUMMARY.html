End-of-Turn (EOT) Detection — Solution Summary

Built from scratch (no pretrained models) for the Plivo EOT assignment.

1. The problem, in one line

At every silence pause inside a user's turn, predict p_eot — the probability the user is actually done talking (vs. just pausing) — using only audio from before that pause (hard causality rule). A live agent then uses these scores plus a short action delay to decide when to respond.

3. Approach

3.1 Why prosody, not ASR/lexical features

No pretrained models or external datasets are allowed, and the hidden test set is mostly Hindi while our labelled dev data is half English. Any lexical/ASR-based feature would be built (implicitly or explicitly) around English text patterns and would transfer poorly. Instead every feature is a language-agnostic acoustic/prosodic cue for turn-taking — the same cues (falling pitch, energy decay, final-syllable lengthening, trailing off into silence vs. staying voiced/rising) show up across languages in conversational speech.

3.2 Features (16 total, all causal)

GroupFeaturesIntuition
Energytail energy, energy decay, energy slope, energy window std Statements trail off in loudness before a real EOT; a hold-pause often cuts off mid-loudness.
Pitchfinal pitch (normalized to the turn's own median pitch), pitch slope over the final voiced stretch, voicing ratio in the last 1.5s Falling pitch → statement-final. Level/rising pitch, or trailing unvoiced sounds → often a continuation or a question the speaker is still building.
Rhythmfinal voiced-run length vs. the turn's average voiced-run length, count of voiced runs, speaking rate so far Final-syllable lengthening is a classic pre-boundary cue independent of language.
Turn contextpause index, elapsed time, count/mean/max duration of already-finished prior pauses in this turn Fully causal (those pauses are over by the time we score the current one) and captures how "deep" into the turn we are.

All feature code lives in eot/features.py. Every function that touches pause_start masks frames to end <= pause_start; pause_end of the current pause is never read by any feature — only predict.py's label-stripping and the scorer ever see it.

3.3 Model

Three candidates (Logistic Regression, RBF-SVM, Gradient Boosted Trees) were compared with 5-fold GroupKFold cross-validation, grouped by turn_id and pooled across both language folders, so no turn's pauses ever leak across train/validation and the model isn't over-fit to one language. Gradient Boosting (shallow trees, depth 2, 150 estimators, lr 0.05, subsampling 0.8) won on CV AUC and was refit on all 200 turns / 496 pauses to produce the shipped eot/model.joblib.

5. Why this beats the status quo

  • Silence-only baseline (naive VAD-style endpointing): AUC ≈ 0.5 by construction — it has no signal, it only benefits from tuning the fixed timeout, which is exactly the ~1600 ms / 850 ms numbers above.
  • Starter train.py (3 weak features: raw tail energy, raw final pitch, context length) scores only marginally above baseline — it was written to be weak on purpose.
  • Our model replaces raw values with relative / turn-normalized cues (pitch relative to the speaker's own median, final voiced run relative to their own average run length, energy decay rather than a static level) plus turn context, and validates model choice with turn-grouped CV across both languages — directly targeting the causal, language-agnostic signal the assignment description calls out ("F0 slope", "final-syllable lengthening", "energy decay rate", "speaking-rate context, position of the pause").
